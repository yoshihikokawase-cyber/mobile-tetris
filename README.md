# Mobile Tetris

HTML / CSS / JavaScript のみで動く、PC・スマホ完全対応版テトリスです。

---

## ファイル構成

```
mobile-tetris/
├── index.html            # メイン HTML
├── style.css             # スタイル（レスポンシブ対応）

├── package.json          # npm スクリプト
├── .gitignore            # Git除外設定
├── tests/
│   └── tetris.spec.js    # E2E テスト
```

## Deploy (Vercel)

1. GitHub にリポジトリを push する
2. [vercel.com](https://vercel.com) でプロジェクトを Import
3. 設定:
   - **Framework Preset**: `Other`
   - **Build Command**: *(空欄のまま)*
   - **Output Directory**: *(空欄のまま / ルートを配信)*
4. `Deploy` をクリック

> 静的ファイル（HTML / CSS / JS）のみで動作するため、ビルド不要です。

## セットアップ・テスト実行

### セットアップ
```bash
cd mobile-tetris
npm install
npx playwright install
```

### ゲーム起動
```bash
npm run start
# → http://localhost:8080 を開く
```

### E2E テスト実行（ヘッドレス）
```bash
npm run test:e2e
```

### E2E テスト実行（UI モード）
```bash
npm run test:e2e:ui
```

### 特定ブラウザのみ実行
```bash
npx playwright test --project=chromium
npx playwright test --project=mobile-chrome
npx playwright test --project=mobile-safari
```

### テストレポート表示
```bash
npx playwright show-report
```

## 操作方法

### PC キーボード

| キー | 操作 |
|------|------|
| `←` / `→` | 左右移動 |
| `↓` | ソフトドロップ（1マス落下） |
| `↑` | 回転 |
| `Space` | ハードドロップ（即座に落下・固定） |
| `P` | 一時停止 / 再開 |
| `R` | リスタート |

### スマホ タッチ操作

| ボタン | 動作 | 備考 |
|--------|------|------|
| ◀ | 左移動 | 長押しで連続移動 |
| ▶ | 右移動 | 長押しで連続移動 |
| ↺ ROTATE | 回転（反時計回り） | SRS ウォールキック対応 |
| ▼ | ソフトドロップ | 長押しで高速落下 |
| ⬇ DROP | ハードドロップ（即落下・固定） | 上部配置で誤操作防止 |

> **ボタン配置の設計思想**: 右ゾーンは「DROP（上）→ ROTATE（下）」の順。
> 親指は自然に下に落ちるため、最多使用の ROTATE が押しやすい位置にある。
> DROP は意識的に上を押す必要があり、誤ハードドロップを防ぐ。

## スコア計算

| 消去ライン数 | 基本点 |
|-------------|--------|
| 1ライン | 100 × レベル |
| 2ライン | 300 × レベル |
| 3ライン | 500 × レベル |
| 4ライン（テトリス） | 800 × レベル |

- ソフトドロップ: 1マス = +1点
- ハードドロップ: 1マス = +2点

## Day9 スマホ UX 改善記録

### 改善前の課題

| 課題 | 内容 |
|------|------|
| 誤ハードドロップ | 回転の直下に DROP があり、親指が流れると即落下 |
| 誤操作しやすい間隔 | 左右ボタンが 12px 間隔で近く、高速操作時に両押し |
| 操作の「もっさり」感 | ソフトドロップの DAS/ARR が左右と同じで反応遅め |
| 視覚的区別なし | 回転と DROP が同サイズで「危険度」が伝わらない |
| スコア折り返し | 10,000 点超えで数字が途中改行されて読みにくい |
| iOS 底部のはみ出し | ノッチ・ホームバー部分にボタンが重なる |
| タッチ入力の不安定さ | touch/mouse イベントが分離していて連打で抜けることがある |

### 実施した修正

**配置・レイアウト**
- DROP ボタンを右ゾーン上部へ移動、ROTATE を下部（親指の自然位置）に配置
- 左右ボタン間隔を 12px → 20px に拡大
- ソフトドロップ行との縦間隔を 12px → 16px に拡大
- 回転↔DROP 間を 18px → 22px に拡大

**ボタンサイズの視覚的階層化**
- ROTATE: 72px → 76px（最頻使用 = 最大サイズ）
- DROP: 72×56px → 64×46px（意識的操作が必要 = 小さく）

**タイミング調整（DAS/ARR）**
- 左右移動: リピート 80ms → 60ms（キビキビした横移動）
- ソフトドロップ: 開始 180ms → 100ms、リピート 80ms → 50ms

**その他**
- iOS safe area: `env(safe-area-inset-bottom)` で底部ボタンを保護
- Pointer Events 統一（`pointerdown/up/cancel/leave`）で入力安定化
- `visualViewport.height` 対応でアドレスバー表示時のサイズズレを修正
- スコア表示: `white-space: nowrap` + `font-variant-numeric: tabular-nums` で折り返し解消

### スマホ確認ポイント

1. **右ゾーン配置**: ↺ ROTATE が下、⬇ DROP が上になっているか
2. **誤 DROP 防止**: 回転を連打しても DROP が発動しないか
3. **左右の押し分け**: 高速横移動で両押しが起きないか
4. **ソフトドロップの速さ**: 長押しですぐ高速落下に移行するか
5. **iPhone ホームバー**: コントロールエリアがホームバーに重ならないか
6. **大きいスコア**: 10,000 点超えでスコアが折り返さないか
7. **横向き**: 端末を横にしてもレイアウトが崩れないか

### 残課題（次回候補）

- **ライン消去アニメーション**: 消えた行が視覚的にわかりにくい
- **ハイスコア保存**: ゲームオーバー後にベストスコアが消える
